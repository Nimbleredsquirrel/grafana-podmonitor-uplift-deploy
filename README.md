# Uplift Model Deployment with Seldon, Prometheus, and Grafana

Trains CatBoost-based uplift models (SoloModel and TwoModels) on the X5 retail dataset, packages them with MLServer, and deploys to a local Kubernetes cluster via Seldon Core. Prometheus scrapes model metrics through a PodMonitor and Grafana visualizes them.

## Project Structure

```
grafana-podmonitor-uplift-deploy/
├── train.py                  # Trains and saves one_model.joblib / two_model.joblib
├── inference.py              # MLServer MLModel implementation (UpliftPredictor)
├── edit_model_settings.py    # Updates model-settings.json with the chosen model file
├── model-settings.json       # MLServer model config (name, implementation, uri)
├── requirements.txt          # Python dependencies
├── Makefile                  # All workflow commands
├── kind-cluster.yaml         # KinD cluster: 1 control plane + 3 workers
├── uplift-deploy.yaml        # Seldon SeldonDeployment manifest
├── podmonitor.yaml           # Prometheus PodMonitor for Seldon pods
└── grafana.yaml              # Grafana Helm values (Prometheus datasource)
```

## Workflow

### 1. Spin up the Kubernetes cluster and install components

```bash
make deploy-all
```

This runs: `kind-cluster` -> `ambassador` -> `seldon-core` -> `prometheus` -> `grafana`

### 2. Train the models

```bash
make train
```

Creates a virtual environment, installs dependencies, and runs `train.py`. Produces `one_model.joblib` (SoloModel) and `two_model.joblib` (TwoModels) in the project root.

### 3. Build and push the Docker image

For the SoloModel:
```bash
make build-solo
make push-image
```

For TwoModels:
```bash
make build-two
make push-image
```

`build-*` targets update `model-settings.json` with the correct model file URI, then call `mlserver build` to package the image as `alph8rd/uplift-predictor`.

### 4. Deploy the model

```bash
make deploy-uplift-predictor
make podmonitor
```

### 5. Access services

```bash
make prom-forward-port    # Prometheus at localhost:9090
make graf-forward-port    # Grafana at localhost:3000 (prints admin password)
```

## Makefile Targets

| Target | Description |
|--------|-------------|
| `deploy-all` | Full cluster and component setup |
| `kind-cluster` | Create KinD cluster (1 control plane, 3 workers) |
| `ambassador` | Install Ambassador API Gateway v6.9.5 |
| `seldon-core` | Install Seldon Core v1.15 |
| `prometheus` | Install kube-prometheus v8.9.1 in seldon-monitoring namespace |
| `grafana` | Install Grafana v6.56.1 with Prometheus datasource |
| `train` | Create venv, install deps, train both models |
| `build-solo` | Update config for SoloModel, build Docker image |
| `build-two` | Update config for TwoModels, build Docker image |
| `push-image` | Push image to Docker Hub |
| `deploy-uplift-predictor` | Apply Seldon deployment manifest |
| `delete-uplift-predictor` | Remove Seldon deployment |
| `podmonitor` | Apply Prometheus PodMonitor |
| `prom-forward-port` | Forward Prometheus to localhost:9090 |
| `graf-forward-port` | Print Grafana password and forward to localhost:3000 |

## Monitoring

The PodMonitor selects pods labeled `app.kubernetes.io/managed-by: seldon-core` across all namespaces and scrapes the `/prometheus` endpoint on the `metrics` port. Grafana is pre-configured with the in-cluster Prometheus URL as its datasource.

## Data

| File | Description |
|------|-------------|
| `data/processed_features.parquet` | Client features from the X5 dataset (generated by train.py) |
| `data/processed_train_data.parquet` | Training targets and treatment flags (generated by train.py) |
| `one_model.joblib` | Trained SoloModel (generated by train.py) |
| `two_model.joblib` | Trained TwoModels (generated by train.py) |
